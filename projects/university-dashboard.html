01. the problem

University classroom management is fragmented — attendance, grades, assignments, and schedules all live in different tools or spreadsheets. Students, faculty, and admins each need different views of the same data.

Key challenge: Building a single platform that serves three distinct user roles — students, faculty, and administrators — each with different permissions, views, and capabilities, while keeping the codebase clean and maintainable.

02. the solution

A PERN stack platform with role-based access control at the API level. Each user role gets a tailored dashboard view while sharing the same underlying data model.

06. lessons learned

  • Role-based access control is best enforced at the middleware level — not the frontend.
  • JWT token expiry and refresh logic is deceptively complex — plan for it from day one.
  • PostgreSQL's relational structure is a much better fit for this use case than MongoDB would have been.
  • Agile sprint-based self-management with GitHub Projects keeps solo projects on track.
  • Designing the database schema before writing a single line of API code saves enormous refactoring time.
